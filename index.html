HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • In general, following the 'separation of concerns' philosophy, stylesheets are to be kept external.
  • <link rel="stylesheet" href="./assets/style.css"> to be placed inside the <head> to add global css.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds all the modified files in the local.
  • git pull origin main: checks whether any changes have been made to the main branch in the interim.
  • git commit -m "message body": commits with said message. Best practices suggest present tense imperative.
  • git push origin branch-name: pushes the local commit to the origin.

JavaScript

  • Arrays are zero-led.
  • Arrays may be initialised very simply - [];
  • JavaScript has two comparisons === strict, and == loose. The single = is for value assignment.
  • Script included in html file via <script src="./assets/script.js"></script>.
  • Script to be included at the end of the body element in html file, presumably to avoid causing delays in loading the page.
